## ADDED Requirements

### Requirement: Elasticsearch textbook chunk index
The system SHALL index canonical textbook chunks into Elasticsearch for teacher question-generation retrieval.

#### Scenario: Textbook chunks are indexed
- **WHEN** the chunk indexer reads the canonical textbook chunk JSONL files
- **THEN** it SHALL create or update Elasticsearch documents containing chunk id, text, source metadata, content hash, source file, and indexed timestamp
- **AND** each document SHALL include an embedding generated by the configured Qwen embedding model.

#### Scenario: Embedding model changes
- **WHEN** the configured embedding model name or vector dimension differs from the existing index metadata
- **THEN** the system SHALL require rebuilding or reindexing the textbook chunk index before treating the index as healthy
- **AND** it SHALL NOT mix vectors produced by different embedding models in one active index.

#### Scenario: Legacy BGE vectors exist
- **WHEN** legacy BGE vectors or pgvector rows are present
- **THEN** the textbook question-generation RAG route SHALL ignore them
- **AND** it SHALL use the configured Qwen/Elasticsearch index for this workflow.

### Requirement: Configured Qwen embedding and rerank retrieval
The system SHALL use configured Qwen-compatible embedding and rerank models for textbook RAG retrieval.

#### Scenario: Query embedding is requested
- **WHEN** the backend prepares a textbook RAG lookup
- **THEN** it SHALL call the configured embedding endpoint with the configured embedding model
- **AND** it SHALL search the active Elasticsearch chunk index with the returned vector.

#### Scenario: Candidate rerank is requested
- **WHEN** Elasticsearch returns a candidate pool for a query section
- **THEN** the backend SHALL call the configured rerank endpoint with the configured rerank model
- **AND** final evidence order SHALL follow rerank scores while preserving Elasticsearch scores and source metadata as diagnostics.

#### Scenario: Retrieval dependency is unavailable
- **WHEN** Elasticsearch, the embedding endpoint, or the rerank endpoint is unavailable
- **THEN** the textbook RAG runtime SHALL be unhealthy
- **AND** teacher question generation SHALL fail closed with a diagnostic stage and reason.

### Requirement: Sectioned point evidence retrieval
The system SHALL retrieve textbook evidence separately for point principle, phenomenon, and safety sections.

#### Scenario: Point has three description sections
- **WHEN** a teacher starts generation for a point with principle, phenomenon, and safety descriptions
- **THEN** the backend SHALL create separate retrieval queries for `principle`, `phenomenon`, and `safety`
- **AND** each query SHALL include the point title, experiment title, chapter, folder path, and that section's description.

#### Scenario: Cross-chapter evidence is relevant
- **WHEN** relevant theory evidence exists outside the selected point's chapter
- **THEN** retrieval SHALL use chapter and experiment as soft boosts rather than mandatory filters
- **AND** it SHALL preserve retrieved source chapter and content type in diagnostics.

#### Scenario: Evidence package is produced
- **WHEN** retrieval succeeds for one or more sections
- **THEN** the evidence package SHALL group final chunks by section
- **AND** it SHALL include chunk ids, text excerpts, source metadata, recall scores, rerank scores, and sufficiency status.

### Requirement: Evidence sufficiency diagnostics
The system SHALL expose enough diagnostics to decide whether generated candidates are properly grounded.

#### Scenario: All selected sections lack usable evidence
- **WHEN** no selected section has evidence above the configured sufficiency threshold
- **THEN** the system SHALL refuse candidate generation
- **AND** it SHALL return a teacher-readable diagnostic instead of creating a publishable candidate.

#### Scenario: Some sections lack usable evidence
- **WHEN** at least one selected section has usable evidence and another selected section lacks evidence
- **THEN** the system MAY generate candidates that cover only the supported sections
- **AND** it SHALL record missing section diagnostics in the candidate or session metadata.

#### Scenario: Diagnostics are inspected
- **WHEN** an administrator or teacher inspects a RAG-backed workbench turn
- **THEN** the system SHALL show retrieval mode, configured model names, index name, query sections, candidate counts, rerank scores, final evidence count, and missing evidence reasons.
